# ExMac-Bridge TODOリスト及びタスク詳細

企画書で定義された4つのフェーズに基づく、実装のためのTODOリストと各タスクの詳細です。

## フェーズ1: コアエンジンのビルドと動作検証
- [ ] **開発環境のセットアップ**
  - Homebrew、Xcode Command Line Toolsの導入確認。
  - Wineビルドに必要な依存ライブラリ（bison, flex, mingw-w64等）のインストール。
- [ ] **カスタムWineのソースコード取得**
  - CrossOver Macなど、Apple Silicon対応が進んでいるWineプロジェクトのソースコードをフォーク・取得する。
- [ ] **Wineのカスタムビルド（軽量化）**
  - macOS環境向けにソースコードからコンパイル。
  - 本アプリに不要な古いWindowsコンポーネントを削減し、軽量なコアエンジンを作成。
- [ ] **CLIでの基本動作検証**
  - ターミナルから手動で `wine target_app.exe` を実行する。
  - 想定通りに描画処理（Quartz経由）や音声出力が行われるか検証する。

## フェーズ2: ブリッジスクリプトの実装
- [ ] **実行ラッパー `launch_wrapper.sh` の作成**
  - 生成された`.app`が起動された際に起点となるシェルスクリプトを作成。
- [ ] **WINEPREFIXの初期化処理**
  - アプリのサンドボックス（`.app`内部構造など指定のディレクトリ）内に仮想Cドライブ環境を自動設定するロジックを実装。
- [ ] **描画エンジンの接続**
  - macOSのディスプレイサーバー(Quartz)とWineの描画エンジンを正しく接続するための環境変数の設定（`DISPLAY`等の扱い）。
- [ ] **エラーハンドリングとネイティブ通知連携**
  - Wine起動時に標準エラー出力をキャッチする。
  - 起動失敗時、macOSネイティブの通知機能（AppleScriptの `display dialog` や、Swiftで作った専用の通知ツールを叩くなど）を利用してユーザーにエラーを分かりやすく表示する。

## フェーズ3: パッケージャーUIの開発 (Swift / AppKit)
- [ ] **Xcodeプロジェクトの作成**
  - 言語: Swift、UIフレームワーク: AppKit にて新規プロジェクトを作成。
- [ ] **ドラッグ＆ドロップUIの実装**
  - メインウィンドウにターゲットとなる `.exe` ファイルを受け付けるドロップエリアを実装。
- [ ] **ファイル解析とリソース抽出**
  - ドロップされた `.exe` のパスを取得。
  - プログラム側でバイナリからアイコンを抽出し、macOS用の `.icns` 形式に変換する処理を検討・組み込む（難しい場合は汎用アイコンを設定）。
- [ ] **カプセル化（`.app` 動的生成）ロジック**
  - `ExMac-Bridge.app` テンプレートとなるフォルダ構造（`Contents/MacOS`, `Resources`, `Frameworks`）を複製。
  - Wineカスタムビルド本体、`launch_wrapper.sh`、および対象の `.exe` を該当ディレクトリにコピー。
  - `Info.plist` の自動生成とメタデータ（アプリ名、バージョン等）の書き換え処理を実装。
- [ ] **システム(Finder)への登録機能**
  - 生成が完了した `.app` パッケージを任意の場所（Applicationsフォルダ等）に出力し、Finder上で認識させる。

## フェーズ4: テストと最適化
- [ ] **ケース1: 軽量な標準Windowsツールの起動テスト**
  - メモ帳や軽いフリーソフトなどで、基礎的なAPI変換が機能するかテスト。
- [ ] **ケース2: DirectX依存のアプリケーションテスト**
  - 3D描画を用いたゲームでの互換性テストと、パフォーマンスの最適化。
- [ ] **ケース3: .NET Framework依存アプリのテスト**
  - 追加でインストールが必要なライブラリやコンポーネントの挙動の確認。
- [ ] **バグフィックス・ブラッシュアップ**
  - 原因不明なクラッシュの修正や、起動速度の向上に向けた調整。
