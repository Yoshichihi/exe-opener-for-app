# ExMac-Bridge (exe-opener-for-app)

## 概要
「ExMac-Bridge」は、Mac環境（macOS）においてWindowsの実行ファイル（.exe）をシームレスに起動するためのネイティブ化ラッパーアプリケーションです。
Wine（互換レイヤー）とAppleのRosetta 2（動的バイナリ変換）という2つの技術を統合することで、重い仮想マシン（Windows OS自体）を立ち上げることなく、Macネイティブのアイコンをダブルクリックする感覚で`.exe`ファイルを実行できる環境を提供します。

## 主な機能
- **ドラッグ＆ドロップ変換**: 動かしたい `.exe` ファイルをアプリにドロップするだけで、バックグラウンド処理により自動で macOSネイティブの `.app` 形式にカプセル化。仮想マシンの起動処理は不要です。
- **軽量・高速な実行**: ホストOS（macOS）とゲストアプリのシステムコール（Windows APIコール）間をリアルタイム翻訳するため、ハードウェア全体をエミュレーションする完全仮想化よりも軽量かつ高速に動作します。
- **Macネイティブな環境の統合**: 実行エラー発生時のMac標準通知（NSAlert）への連携機能を備え、macOSのデザイン・機能に馴染む使用感を提供します。

## アーキテクチャと動作モジュール
本アプリケーションは、以下の3つの主要なモジュールから構成されています。

1. **GUIフロントエンド (Swift / AppKit)**
   ユーザーからの入力を受け取るネイティブUIです。ドロップされたファイルをもとに、専用の `.app` バンドルディレクトリツリーを動的に生成します。
2. **実行制御・ブリッジモジュール (Shell Script)**
   Wineの仮想Cドライブ（WINEPREFIX）の初期化、macOSディスプレイサーバー(Quartz)との描画セッションの接続、および実行ハンドリングを司ります。
3. **コア変換エンジン (Wine Custom Build)**
   Windowsの機械語命令とAPI呼び出しを傍受し、macOSのネイティブAPIへ即座に翻訳する本システムの心臓部です（Apple Silicon向けの対応が強化された不要コンポーネントのないカスタムビルドを使用します）。

### 生成される .app パッケージの構成例
本アプリによって変換・生成されたパッケージ構造は以下のようになります。
```text
(UserApp).app/
 └── Contents/
      ├── Info.plist
      ├── MacOS/
      │    ├── ExMac-Bridge        # バックグラウンド制御バイナリ
      │    └── launch_wrapper.sh   # 起動用のブリッジスクリプト
      ├── Resources/
      │    ├── AppIcon.icns        # 自動付与されるアプリアイコン
      │    └── target_app.exe      # ドロップされた元のWindowsアプリ
      └── Frameworks/
           └── Wine.framework      # API変換用コアエンジン(Wine)
```

## 開発環境・必須要件
開発およびソースコードのビルドには以下の環境が必要です。
- **OS**: macOS (Apple Silicon M1/M2/M3推奨、Rosetta 2の利用)
- **IDE**: Xcode (Swift AppKit用)
- **パッケージマネージャ**: Homebrew (Wineコンパイル用依存ライブラリ導入のため)

*(※詳細な環境構築手順や開発ロードマップについては、同ディレクトリの `PLAN.md` および `TODO.md` をご参照ください。)*

## 留意事項
本プロジェクトが用いる「互換レイヤー技術」と計算機科学上の「停止性問題」により、事前動作の100%の予測および、非公開Windows APIを使う強固なアンチチートやDRM技術を含む実行ファイルの動作保証はできません。本ツール利用による不具合等については技術的免責事項となります。
